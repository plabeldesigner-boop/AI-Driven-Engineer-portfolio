Horizontal Scaling with Redis Pub/Sub Clusters

Published on May 15, 2026 by Mutasim Billah

Monolithic architectures provide excellent write/read speeds and simple project structures. However, as user traffic scales, a single server instance becomes a bottleneck. To scale horizontally, we must decouple server states.

Stateless session routing is the first step. By utilizing database-backed session stores (like MongoDB via connect-mongo), user authentication sessions can be resolved regardless of which server instance receives the HTTP request.

The real challenge lies in persistent connections, such as WebSockets. If User A is connected to Server Node 1, and User B is connected to Server Node 2, they cannot communicate directly through memory.

To solve this, we configure a **Redis Pub/Sub adapter** for Socket.IO. When a message is sent, the local server publishes the payload to a Redis channel. Every other server node, listening to Redis, receives the event and broadcasts it to its local connected clients.

This ensures seamless delivery of real-time events (including read receipts, typing notifications, and live status states) across decentralized multi-process clusters without bottlenecking application memory.
